# Проект по курсу "Веб-программирование на Python (Django)"

Этот репозиторий содержит выполненные практические работы по фреймворку Django. Проект представляет собой простое веб-приложение, демонстрирующее ключевые возможности Django, от маршрутизации и представлений до работы с шаблонами, формами и моделями баз данных.

## О структуре репозитория и объединении работ

Практические работы с 12 по 15 были выполнены в рамках единого проекта и находятся в одной ветке репозитория. Такое решение было принято осознанно, так как данные задания представляют собой последовательное и логическое развитие одного веб-приложения:

1.  **ПРЗ 12 ("Представления и маршрутизация")** заложила основу проекта: были созданы функции-представления (логика) и настроены URL-маршруты для доступа к ним.

2.  **ПРЗ 13 ("Шаблоны")** стала следующим критически важным шагом. Она позволила "оживить" представления, создав для них визуальное отображение в виде HTML-страниц. Были изучены наследование шаблонов (`{% extends %}`), подключение статических файлов (CSS, изображения) и динамическая подстановка данных.

3.  **ПРЗ 14 ("Работа с формами")** расширила функциональность, добавив возможность интерактивного взаимодействия с пользователем. Формы, созданные в этой работе, рендерятся внутри шаблонов (из ПРЗ 13) и позволяют отправлять данные на обработку в представления (из ПРЗ 12).

4.  **ПРЗ 15 ("Работа с моделями")** стала логическим завершением цикла, добавив уровень данных (backend). Модели позволили сохранять в базу данных информацию, полученную из форм (ПРЗ 14), и выводить ее на страницах, отображаемых с помощью шаблонов (ПРЗ 13).

Таким образом, разделение этих заданий на разные ветки нарушило бы целостность приложения. Объединенный подход позволяет наглядно увидеть, как все основные компоненты Django (Models, Views, Templates, Forms, URLs) работают вместе для создания полноценного веб-приложения.

## Инструкция по запуску и проверке

Для проверки работоспособности проекта выполните следующие шаги.

### 1. Предварительные требования
*   Установленный **Python 3.8** или выше.
*   Установленная система контроля версий **Git**.

### 2. Клонирование и настройка окружения

```bash
# 1. Клонируйте репозиторий на свой компьютер
git clone <URL-вашего-репозитория>

# 2. Перейдите в папку с проектом
cd <название-папки-проекта>

# 3. Создайте виртуальное окружение
python -m venv venv

# 4. Активируйте виртуальное окружение
# Для Windows:
venv\Scripts\activate
# Для macOS / Linux:
source venv/bin/activate

# 5. Установите все необходимые зависимости (в данном случае только Django)
pip install django
```

### 3. Настройка базы данных

```bash
# 1. Примените миграции для создания таблиц в базе данных
python manage.py migrate
```

### 4. Наполнение базы данных (важно!)

Чтобы страницы с данными не были пустыми, необходимо добавить несколько записей в базу.

```bash
# 1. Запустите интерактивную оболочку Django
python manage.py shell

# 2. Внутри оболочки выполните следующий Python-код для создания нескольких студентов:
from myapp.models import Student

Student.objects.create(last_name="Иванов", first_name="Иван", patronymic="Иванович", birth_year=2003, group_number="ИВТ-101")
Student.objects.create(last_name="Петров", first_name="Петр", patronymic="Петрович", birth_year=2004, group_number="ПИН-102")
Student.objects.create(last_name="Сидорова", first_name="Анна", patronymic="Павловна", birth_year=2003, group_number="ИВТ-101")
Student.objects.create(last_name="Смирнов", first_name="Алексей", patronymic="Сергеевич", birth_year=2005, group_number="БИН-103")

# 3. Выйдите из оболочки (Ctrl+Z и Enter для Windows, Ctrl+D для macOS/Linux)
```

### 5. Запуск сервера

```bash
# Запустите сервер для разработки
python manage.py runserver
```
Сервер будет доступен по адресу `http://127.0.0.1:8000/`.

## Проверка работоспособности (URL для тестирования)

После запуска сервера откройте браузер и перейдите по следующим адресам для проверки выполненных заданий:

*   **Главная страница:**
    *   `http://127.0.0.1:8000/`

*   **ПРЗ 12 - Представления и маршрутизация:**
    *   `http://127.0.0.1:8000/user/admin/` — передача параметра `username` из URL.
    *   `http://127.0.0.1:8000/archive/2024/11/` — передача нескольких параметров (`year`, `month`).
    *   `http://127.0.0.1:8000/search/?q=myquery&category=books` — обработка GET-параметров из строки запроса.

*   **ПРЗ 13 - Шаблоны и статические файлы:**
    *   `http://127.0.0.1:8000/statfiles/` — демонстрация подключения статических файлов (CSS и изображения).
    *   Обратите внимание, что **все страницы проекта** (например, `/students/`, `/userdata/`) используют базовый шаблон, демонстрируя механизм **наследования шаблонов**.

*   **ПРЗ 14 - Работа с формами:**
    *   `http://127.0.0.1:8000/formhtml/` — простая HTML-форма с обработкой во view.
    *   `http://127.0.0.1:8000/userdata/` — форма, созданная с помощью `Django Forms`, с валидацией данных. Попробуйте отправить пустую форму, чтобы увидеть сообщения об ошибках.

*   **ПРЗ 15 - Работа с моделями:**
    *   `http://127.0.0.1:8000/students/` — на этой странице вы увидите таблицу со всеми студентами, которые были добавлены в базу данных на шаге 4.
